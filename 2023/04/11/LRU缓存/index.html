LRU缓存

LRU缓存

1.题目

  • 题目来源：https://leetcode.cn/problems/lru-cache/

  • 设计并实现一个满足LRU(最近最少使用)缓存约束的数据结构，实现LRUCache类：

    • LRUCache(int capacity)以正整数作为容量capacity初始化LRU缓存

    • int get(int key) 如果关键字key存在于缓存中，则返回关键字的值，否则返回-1

    • void put(int key, int value) 如果关键字key已经存在，则变更其数据值value

      如果不存在，则向缓存中插入该组key-value

      如果插入操作导致关键字数量超过capacity，则应该逐出最久未使用的关键字

      函数get和put必须以O(1)的平均时间复杂度运行

2.分析

  • 首先需要明白LRU是什么？

    LRU缓存是最近最少使用缓存，是一种缓存策略，根据数据最近被使用的时间，将最近最少使用的数据从缓存中删除

    在每次访问缓存中的数据时，该数据的状态都会被更新，表示它是最近被使用的

    当缓存中的数据达到设定的最大容量的时候，就会将最早被访问到的数据从缓存中移除

  • 理解了LRU原理之后，分析需要用到的数据结构：哈希表+双向链表

    在Java语言中有封装好的数据结构：LinkedHashMap

    所以，这道题分成两种实现方式：

    1. 使用Java语言的LinkedHashMap
    2. 手写双向链表+哈希表实现

3.LinkedHashMap实现

3.1 LinkedHashMap是什么？和HashMap的区别？

  • LinkedHashMap继承自HashMap，并且在HashMap的基础上增加了一个双向链表来保持数据的插入顺序
  • 需要保证元素顺序和插入顺序一致的时候，使用LinkedHashMap
  • 如果只需要快速插入和查找元素，使用HashMap即可

3.2 逻辑思路

  • LRUCache(int capacity)需要一个类变量，初始化cap

  • int get(int key)函数判断key是否存在缓存，因此使用if判断

    • if(不包含key) 返回-1

    • 包含key的话，在获取这个key对应的value的时候，需要使其变为最近访问的缓存

      即将其value值记录下来，然后删除该key值缓存，重新插入LinkedHashMap

  • void put(int key, int value)函数为新增函数，首先需要判断是否已经存在

    • if(包含key)将其删除，然后重新插入LinkedHashMap

    • else if(不包含key) 即新增元素，此时判断容量if(长度>=容量)，需要将最早的元素删除

      删除最早元素，可以借助LinkedHashMap的keySet遍历，即.keySet().iterator().next()方法，然后插入元素

    • else if(容量没满 && 不存在) 直接插入元素即可

4.手写双向链表+哈希表实现

4.1 HashMap是什么？双向链表是什么？

  • HashMap是常用的Java集合类，采用哈希表数据结构实现，使用键值对存储元素，用于元素的快速查找，插入和删除
  • 双向链表在链表的基础上增加了一个指针域，指向前一个节点

4.2 逻辑思路

  • 双向链表的节点长什么样子？

    节点Node有数据域和指针域，数据域存放key值和value，同时两个指针，pre和next，分别指向前一个节点和后一个节点，即节点(pre|key|value|next)，双向链表(head<->[key,value]<->tail)

  • HashMap存什么？

    HashMap的key值存放双向链表的key，value存放节点Node，即<key,Node>

  • 需要的实现的功能？

    • 插入新元素：采用头插法，将最近使用的节点插入双向链表
    • 删除某一个元素：改变前一元素的next指针和后一元素的pre指针
    • 删除最早元素：最早元素就是尾指针前一元素，将其获取然后删除

  • 写完方法后，实现的逻辑同LinkedHashMap
